Introduo

A Secretaria de Estado de Fazenda de Mato Grosso passa a publicar bimestralmente o Boletim Fiscal, em cumprimento Lei de Diretriz Oramentria (LDO), com o objetivo de ampliar a transparncia fiscal, avaliar o desempenho das receitas e dos gastos pblicos e fomentar as discusses acerca das finanas pblicas.

O documento traz dados e anlises sobre informaes pertinentes s conjunturas econmica e fiscal do estado, alm do monitoramento dos indicadores e metas fiscais que auxiliam o gerenciamento das contas pblicas estaduais.

Tem sido cada vez mais comum encontrar informaes e anlises sobre as receitas e despesas do governo na imprensa, nos discursos de polticos e autoridades, e at nas conversas do dia-a-dia. Em geral, comparam-se receitas e despesas, fala-se em supervit ou dficit, analisa-se o crescimento das despesas, a sua composio etc. Mas como as receitas e as despesas oramentrias da Unio de fato se relacionam?

Ou seja, o que pretendemos responder, talvez de forma indita, a seguinte pergunta: que receitas na prtica financiam que despesas? Para isso, construmos uma visualizao, agrupando as receitas e despesas em categorias, com o intuito de simplificar a extensa classificao oramentria adotada no setor pblico brasileiro, e estimamos a relao entre cada uma dessas receitas e despesas. Os critrios tcnicos adotados nesses agrupamentos esto descritos na seo de Metodologia, abaixo.

Esse tipo de visualizao pode ajudar a iluminar algumas questes que surgem com frequncia a respeito da gesto de finanas pblicas no Governo Federal, por exemplo:

  • verdade que recursos dos tributos (impostos, contribuies e taxas) so gastos para pagar os juros da dvida?

  • Se h um dficit no Regime Geral de Previdncia Social, como ele financiado?

  • Se o governo vem apresentando dficit nas contas, de onde vem o dinheiro para pagar suas despesas?

  • O que feito com o dinheiro obtido pelo governo com a emisso e venda de ttulos pblicos, tais como os do Tesouro Direto?

Como as receitas foram classificadas

Existem diversas formas possveis de se agruparem as receitas oramentrias. Para este exerccio, procuramos destacar certos agrupamentos de receitas, tais como as emisses de ttulos pblicos, as receitas do INSS, as receitas de impostos etc.

A classificao por natureza de receitas utiliza uma estrutura de codificao baseada num cdigo numrico de 8 dgitos, em que as posies dos dgitos constituem classificadores intermedirios, da seguinte forma:

Estrutura da Natureza de Receita. Fonte: MTO 2018

Para esta anlise, as receitas foram agrupadas no seu terceiro nvel, o nvel de Espcie. Alm disso, os recursos do Regime Geral de Previdncia Social foram tratados de forma separada, por meio do classificador unidade oramentria.

Assim, as receitas foram agrupadas da seguinte forma:

Para as despesas, foi adotada uma classificao baseada naquela utiliza no captulo Estrutura do Gasto Pblico no Brasil, de autoria de Mansueto Almeida, do livro A Reforma Esquecida, acrescentando-se as despesas da dvida pblica e as transferncias a estados e municpios. Essa classificao estabelece um recorte das despesas pblicas da seguinte forma:

rec <- read_excel("./dados/Receita.xlsx", sheet = 1, col_names = TRUE)
des <- read_excel("./dados/Despesa.xlsx", sheet = 1, col_names = TRUE)

names(rec) <- c("nr", "fte", "fte_nome", "rec")
names(des) <- c("fte", "fte_nome", "nd", "des")

rec <- rec %>%
  group_by(nr) %>%
  mutate(rec = rec,  #/1000000
         subtot_rec = sum(rec),
         per_rec_fte = rec/sum(rec)) # calcula os % de distrib. de cada rec nas ftes

des <- des %>%
  group_by(fte) %>%
  mutate(des = des, #/1000000
         per_fte_des = des/sum(des)) # calcula os % de distrib. de fte nas desps



# tenho uma lista de fontes

matriz <- rec %>%
  full_join(des, by = "fte") # pelo cdigo da fonte, pq vai que o nome da fonte muda...

# calcula o % de dist. de cada rec nas desps
# calcula ento o tamanho de cada link, partir do subtotal de cada receita. ignorando se for menor que um bilhao
### a fazer: remover as linhas com um anti_join, para no contaminar a informao do hoover (outgoing, ingoing count)

matriz <- matriz %>% 
  mutate(p = per_rec_fte * per_fte_des, 
         # ramo = ifelse((round(subtot_rec * p,0)<1000000000),0,round(subtot_rec * p,0))) %>%
         ramo = round(subtot_rec * p,0)) %>% 
  group_by(nr,nd) %>%                        
  summarise(p = sum(p),
            ramo = sum(ramo)) %>%
  select(nr,nd,p,ramo)

# limpeza

tamanho_critico <- 1e1 
ramos_a_limpar <- matriz %>% filter(ramo%
  anti_join(ramos_a_limpar)

# relacao unica dos rotulos de receita e despesa:
rotulos <- c(unique(matriz$nr),unique(matriz$nd))

# conta os ns e gera sequencia numerica a partir de zero
num_nos <- length(rotulos)
nos <- 0:(num_nos-1)

# cria tabelinha para numerar os nos
tab_aux <- data.frame(rotulos, nos)

# incorpora os nmeros dos nodes na matriz, para a receita... e para a despesa.
matriz <- matriz %>%
  left_join(tab_aux, by = c("nr" = "rotulos")) %>%   
  left_join(tab_aux, by = c("nd" = "rotulos"), suffix = c("_rec","_desp"))



# DEFINIES DAS CORES
library(RColorBrewer)

azul_STN <- "#1f476a"
cor_divida_transluc <- "rgba(186, 57, 23, 0.7)"
cor_RPPS_transluc <- "rgba(31, 71, 106, 0.7)"
cor_divida <- "#BA3917"
cor_RPPS <- azul_STN
cor_Pessoal <- "#01665E"

nclr = 9
#pclr <- brewer.pal(nclr, "Greens")#[5:9]# " tons de cores comenando em 4 at 9"
cor_padrao_despesas <- c(brewer.pal(nclr, "Greens"))



qde_rec_destacadas <- 1
qde_cores_extras <- length(unique(matriz$nr)) - qde_rec_destacadas

paleta_STN <- brewer.pal(qde_cores_extras, "Set3")
#paleta_STN <- c("#F8AC08","#CEC806","#96C11E","#028063","#149339","#4E857E","#6E287C")

paleta_STN_rgb <- col2rgb(paleta_STN)

# essa funo gera uma matriz com os componentes RGB.
# precisamos fornecer as cores no formato "rgb(r,g,b)"
# alm disso, para os ramos,  interessante que as cores sejam translcidas, 
# ento vamos acrescentar um componente de transparncia

paleta_STN_transluc <- NULL
for (i in 1:dim(paleta_STN_rgb)[2] ) {
  paleta_STN_transluc <- c(paleta_STN_transluc, paste("rgba(", paleta_STN_rgb[1,i],", ",  paleta_STN_rgb[2,i],", ",  paleta_STN_rgb[3,i],", 0.7)", sep = ""))
}

# rotininha para povoar as cores dos ramos e dos nos, conforme as receitas

receitas <- unique(matriz$nr)

cores_ramos <- NULL
cores_nos <- NULL
k = 1

for (i in 1:length(receitas)) {
  if (receitas[i] == "Contribuies e outras receitas do RPPS") {
    cores_ramos <- c(cores_ramos, cor_RPPS_transluc)
    cores_nos <- c(cores_nos, cor_RPPS)
  }
  else if (receitas[i] == "Contribuies Econmicas") {
    cores_ramos <- c(cores_ramos, cor_divida_transluc)
    cores_nos <- c(cores_nos, cor_divida)
  }
  else {
    cores_ramos <- c(cores_ramos, paleta_STN_transluc[k])
    cores_nos <- c(cores_nos, paleta_STN[k])
    k <- k + 1 # o certo seria fazer algo modular, caso sejam mais receitas que cores
  }
}

## povoando as cores das despesas... (com um mtodo mais elegante do que um loop)
despesas <- unique(matriz$nd)
cores_despesas <- rep(cor_padrao_despesas, length(despesas))

cores_despesas[despesas == "Amortizao da Dvida" | despesas == "Juros"] <- cor_divida

cores_despesas[despesas == "Benefcios Previdencirios RPPS e Assistncia Social"] <- cor_RPPS

cores_despesas[despesas == "Pessoal"] <- cor_Pessoal

## fechando a lista de cores dos nos

cores_nos <- c(cores_nos, cores_despesas)

## tabelinha para usar de referncia no preenchimento da matriz

tabelinha_cores_ramos <- data.frame(receitas, cores_ramos)

## preenchendo as cores dos ramos na matriz a partir dessa tabelinha

matriz <- matriz %>%
  left_join(tabelinha_cores_ramos, by = c("nr" = "receitas"))

## sem dvida

# qual o numero do no de juros
no_juros <- unique(matriz$nos_desp[which(matriz$nd=="Juros")])

# qual o numero do no de amortizao da dvida
no_amort <- unique(matriz$nos_desp[which(matriz$nd=="Amortizao da Dvida")])

matriz_semdivida <- matriz %>%
  mutate(nd = replace(nd, nd=="Juros", "Dvida"),
         nd = replace(nd, nd=="Amortizao da Dvida", "Dvida"),
         nos_desp = replace(nos_desp, nd=="Dvida", min(no_juros,no_amort))) %>%
  #filter(!(nr == "Emisses de ttulos" & nd == "Dvida")) %>% # ateno a esse filtro!
  group_by(nr,nd,nos_rec,nos_desp,cores_ramos)%>%
  summarize(p = sum(p),
            ramo = sum(ramo))

rotulos_divida <- c(unique(matriz_semdivida$nr),unique(matriz_semdivida$nd))

cor_RPPS_transluc <- "rgba(31, 71, 106, 0.7)"
cor_divida <- "#BA3917"
cor_RPPS <- azul_STN
cor_Pessoal <- "#01665E"

nclr = 12
#pclr <- brewer.pal(nclr, "Greens")#[5:9]# " tons de cores comenando em 4 at 9"
cor_padrao_despesas <- c(brewer.pal(nclr, "Paired"))



qde_rec_destacadas <- 1
qde_cores_extras <- length(unique(matriz_LOA$nr)) - qde_rec_destacadas

paleta_STN <- brewer.pal(qde_cores_extras, "Set3")
#paleta_STN <- c("#F8AC08","#CEC806","#96C11E","#028063","#149339","#4E857E","#6E287C")

paleta_STN_rgb <- col2rgb(paleta_STN)

# essa funo gera uma matriz com os componentes RGB.
# precisamos fornecer as cores no formato "rgb(r,g,b)"
# alm disso, para os ramos,  interessante que as cores sejam translcidas, 
# ento vamos acrescentar um componente de transparncia

paleta_STN_transluc <- NULL
for (i in 1:dim(paleta_STN_rgb)[2] ) {
  paleta_STN_transluc <- c(paleta_STN_transluc, paste("rgba(", paleta_STN_rgb[1,i],", ",  paleta_STN_rgb[2,i],", ",  paleta_STN_rgb[3,i],", 0.7)", sep = ""))
}

# rotininha para povoar as cores dos ramos e dos nos, conforme as receitas

receitas_LOA <- unique(matriz_LOA$nr)

cores_ramos <- NULL
cores_nos <- NULL
k = 1

for (i in 1:length(receitas_LOA)) {
  if (receitas_LOA[i] == "Contribuies ao Plano Financeiro e Previdencirio do RPPS") {
    cores_ramos <- c(cores_ramos, cor_RPPS_transluc)
    cores_nos <- c(cores_nos, cor_RPPS)
  }
  else if (receitas_LOA[i] == "Contribuies Econmicas") {
    cores_ramos <- c(cores_ramos, cor_divida_transluc)
    cores_nos <- c(cores_nos, cor_divida)
  }
  else {
    cores_ramos <- c(cores_ramos, paleta_STN_transluc[k])
    cores_nos <- c(cores_nos, paleta_STN[k])
    k <- k + 1 # o certo seria fazer algo modular, caso sejam mais receitas que cores
  }
}

## povoando as cores das despesas... (com um mtodo mais elegante do que um loop)
despesas_LOA <- unique(matriz_LOA$nd)
cores_despesas <- rep(cor_padrao_despesas, length(despesas_LOA))

cores_despesas[despesas_LOA == "Amortizao da Dvida" | despesas_LOA == "Juros"] <- cor_divida

cores_despesas[despesas_LOA == "Previdncia Social (RPPS)"] <- cor_RPPS

cores_despesas[despesas_LOA == "Pessoal e Encargos Sociais"] <- cor_Pessoal

## fechando a lista de cores dos nos

cores_nos <- c(cores_nos, cores_despesas)

## tabelinha para usar de referncia no preenchimento da matriz

tabelinha_cores_ramos <- data.frame(receitas_LOA, cores_ramos)

## preenchendo as cores dos ramos na matriz a partir dessa tabelinha

matriz_LOA <- matriz_LOA %>%
  left_join(tabelinha_cores_ramos, by = c("nr" = "receitas_LOA"))

## sem dvida

# qual o numero do no de juros
no_juros <- unique(matriz_LOA$nos_desp[which(matriz_LOA$nd=="Juros")])

# qual o numero do no de amortizao da dvida
no_amort <- unique(matriz_LOA$nos_desp[which(matriz_LOA$nd=="Amortizao da Dvida")])

matriz_semdivida <- matriz_LOA %>%
  mutate(nd = replace(nd, nd=="Juros", "Dvida"),
         nd = replace(nd, nd=="Amortizao da Dvida", "Dvida"),
         nos_desp = replace(nos_desp, nd=="Dvida", min(no_juros,no_amort))) %>%
  #filter(!(nr == "Emisses de ttulos" & nd == "Dvida")) %>% # ateno a esse filtro!
  group_by(nr,nd,nos_rec,nos_desp,cores_ramos)%>%
